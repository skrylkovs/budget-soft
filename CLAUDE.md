# Правила проекта BUDGET SOFT (prototype-main)

## URL существующих страниц неизменны
- Реестр всех URL — [urls.json](urls.json). Это источник истины по адресам страниц.
- **Никогда не меняй адрес существующей страницы**: не переименовывай и не удаляй её директорию,
  не трогай путь `<dir>/index.html`. На эти URL могут вести внешние ссылки, поисковая индексация и закладки.
- Менять можно только **содержимое** страницы, а не её адрес.
- При добавлении **новой** страницы — создай `<новый-dir>/index.html` и добавь запись в `urls.json`,
  существующие записи не трогай.
- Если задача требует изменить адрес существующей страницы — сначала предупреди и спроси подтверждение,
  и предложи редирект со старого URL вместо простого переименования.

## Структура URL
Сайт статический, «красивые» URL: каждая страница — `<dir>/index.html`, отдаётся как `/<dir>/`.

## Sitemap и `lastmod`
- **Никогда не обновляй `lastmod` массово у всех URL при генерации сайта, сборке или деплое.**
- Меняй `lastmod` только у новой страницы или у страницы, содержимое которой действительно существенно изменилось.
- Для неизменённых страниц сохраняй прежнее значение `lastmod`; не подставляй текущую дату автоматически.
- Изменения общих стилей, скриптов, меню, футера или процесса сборки сами по себе не являются причиной обновлять `lastmod` всех страниц.
- Генераторы `sitemap.xml` должны хранить или вычислять дату отдельно для каждого URL, а не использовать одну дату запуска генератора.
- При существенном изменении страницы обнови поле `lastmod` только в её записи в `urls.json` (`YYYY-MM-DD`). Для записи без этого поля генератор сохраняет прежнюю дату из `sitemap.xml`, а для нового URL ставит дату создания.
- Перед коммитом изменений `sitemap.xml` проверь diff: даты должны измениться только у реально затронутых страниц.
