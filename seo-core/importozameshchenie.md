  миграция без простоя → передача прав). Целевое действие — заявка на консультацию/аудит.
- **Аудитория:** ИТ-директор, ИТ-руководитель и закупщик в компании, у которой
  отозваны лицензии или которая обязана перейти на реестровое ПО (КИИ, госсектор,
  крупный B2B со связями с госзаказом).
- **Рынок/регион:** Россия (федеральное продвижение).
- **Устройства:** все.
- **Дата анализа:** 19.07.2026.

## Методика сбора

- **Маркерные формулировки:** `импортозамещение ПО`, `переход на российское ПО`,
  `миграция с зарубежного ПО на отечественное`.
- **Проверенные маски (14):** импортозамещение по; импортозамещение программного
  обеспечения; переход на российское по; миграция с зарубежного по; импортозамещение 1с;
  аудит импортозамещения; реестр отечественного по; миграция с sap на 1с; замена
  office 365; импортозамещение erp; миграция на postgresql; астра линукс миграция;
  импортозамещение сайта; отечественное по для бизнеса.
- **Критерий остановки расширения:** последние четыре маски дали либо пустой результат
  («астра линукс миграция», «отечественное по для бизнеса» — «ничего не нашлось»), либо
  чужой интент (госпорталы, биржа импортозамещения) — смысловое насыщение достигнуто.
- **Нормализация:** регистр приведён к нижнему, дубли схлопнуты; исходные формулировки
  сохранены. Критично: «по» в запросах — омоним («ПО»-софт vs предлог «по»).
- **Интент:** проверялся по кластеризации Вебмастера, составу уточнений Вордстата и
  органической выдаче Яндекса (Москва, lr=213, desktop, 19.07.2026) по шести
  формулировкам.
- **SERP-кластеризация:** не использовалась (пересечение URL не считалось); выдача
  применялась для проверки интента и отбора конкурентов.
- **Ограничения:** запросная статистика по целевому URL отсутствует (страница в индексе,
  но не ранжируется), поэтому приоритеты опираются на спрос и интент, а не на
  фактические показы. Сбор выполнен в один заход 19.07.2026 — сезонность не учтена;
  тема привязана к регуляторному календарю.

## Рекомендации по странице

1. **Title / H1 / Description.**
   Сейчас: Title «Импортозамещение — BUDGET SOFT» (два значимых слова), **H1 —
   «Разрабатываем ПО любой сложности для автоматизации бизнеса»**, то есть общий хиро
   сайта, вообще не про тему; тематический заголовок понижен до H2. Это главная
   он-пейдж-ошибка страницы.
   Предлагаемое направление: H1 — цельная формулировка вокруг «Импортозамещение
   программного обеспечения» с указанием сути работ (переход на российский стек без
   простоя); Title — с добавлением конкретики (аудит, дорожная карта, миграция);
   Description — с дедлайнами и форматом работ. Точные строки — на шаге контент-документа.
   Метатекст не обязан отображаться в сниппете дословно.
2. **Первый экран.** Должен сразу отвечать: с чего уходим, на что переходим, за какой
   срок, что получаем на выходе (права, исходники, документация). Сейчас первый экран
   несёт общий слоган разработки ПО.
3. **Регуляторный блок — главный контентный пробел.** Весь объём спроса сидит в
   «требования / указ 166 / ПП 1236 / ПП 1478 / сроки / реестр», а страница не отвечает
   ни на один вопрос. Нужен честный блок «что требует закон и к какому сроку» с опорой
   на документы, без юридических гарантий.
4. **Цена / процесс / FAQ.** Нужны: этапы (аудит зависимостей → дорожная карта → пилот →
   миграция → гиперопека), диапазон цены и сроков, FAQ по регуляторике, рискам и
   гарантиям непрерывности. Цены — только согласованные, иначе маркер `[≈ согласовать]`.
5. **Матрица «что на что меняем».** Конкретные миграции (Office 365, 1С на PostgreSQL,
   Oracle/MS SQL → PostgreSQL, SAP → 1С:ERP, СЭД, BI, RPA) — самая коммерчески живая
   часть темы и естественная точка перелинковки на профильные услуги.
6. **Перелинковка:** `/uslugi/vnedrenie-1c/`, `/tehnologii/1c/`,
   `/uslugi/razrabotka-erp-sistem/`, `/uslugi/sistemy-dokumentooborota/`,
   `/uslugi/bi-big-data/`, `/uslugi/vnedrenie-rpa/`, `/uslugi/zashchita-kii/`,
   `/uslugi/administrirovanie-serverov/`, `/uslugi/modernizaciya-legacy/`.
   Границу с `modernizaciya-legacy` держать явной: там — техническая модернизация
   собственного legacy, здесь — уход от зарубежного вендора и регуляторика.
7. **Чего не делать.** Не таргетировать `реестр отечественного по` (навигационный) и
   промышленное импортозамещение; не обещать лицензируемых работ по КИИ; не выдумывать
   субсидии и сроки; не собирать Title из перечисления ключей.
