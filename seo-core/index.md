## Методика сбора

- **Маркерные формулировки:** `разработка ПО на заказ`, `заказная разработка (программного
  обеспечения)`, `компания по разработке ПО`.
- **Проверенные маски (12):** разработка ПО на заказ; заказная разработка; заказная разработка
  программного обеспечения; разработка программного обеспечения (без квалификатора); разработка ПО
  под ключ; разработка программного обеспечения на заказ; компания по разработке программного
  обеспечения; заказать разработку программного обеспечения; создание программного обеспечения на
  заказ; разработка софта на заказ; разработка программ на заказ; стоимость разработки программного
  обеспечения.
- **Критерий остановки расширения:** синонимы «софт», «создание», «заказать» дали единицы запросов,
  а вкладка «Похожие» — только нерелевантный шум; дальнейшие лексемы уводят в конкретные услуги
  (место которых — `/uslugi/*`). Смысловое насыщение зонтика достигнуто.
- **Нормализация:** регистр приведён; Вордстат лемматизирует «ПО» как предлог «по» — операторная
  проверка отделяет коммерческую фразу от «по»-предложных вариантов; исходные формулировки сохранены.
- **Интент:** проверен по органике Яндекса 17.07.2026 (Россия, lr=225) и 12.07.2026 (Москва, lr=213),
  desktop; типы документов — коммерческие страницы услуг + агрегаторы/карты + статьи.
- **SERP-кластеризация:** отдельная hard/soft-кластеризация по порогу пересечений не проводилась;
  интент оценён качественно по составу топ-12 (детальный разбор 15 конкурентов — в `seo-pages/index.md`).
- **Ограничения:** у URL нет исторической статистики (новый сайт); инструмент «Подбор запросов и
  анализ рынка» недоступен; SERP снят на desktop.

## Рекомендации по странице

1. **Title / H1 / Description.**
   - Title (текущий «Разработка ПО на заказ — BUDGET SOFT | Заказная разработка программного
     обеспечения») по данным ядра **валиден** — оба маркера подтверждены Вордстатом; менять ради
     цены не обязательно, но допустимо добавить ценовой/скоростной якорь (у purrweb в топе цена
     уже вынесена в Title). Не превращать в перечень синонимов.
   - **H1 сменить** со слогана «Разрабатываем ПО любой сложности…» на формулировку спроса:
     «Разработка программного обеспечения на заказ» (самый чистый маркер: broad 120 / точная 53).
     ⚠️ Тот же hero-H1 сейчас на всех служебных страницах (`/stoimost/`, `/sroki/`, `/o-nas/`…) —
     менять H1 только на главной, иначе страницы получат одинаковый H1 под один интент.
   - Description — включить «заказная разработка ПО», «на заказ», ценовой/скоростной факт; это
     не перечень ключей и может не попасть в сниппет дословно.
2. **Первый экран и предложение.** Сразу ясно: заказная разработка ПО для бизнеса (зонтик), цифры
   компании (60+, 17 лет), адресация СМБ. Актор-интент («компания-разработчик») усилить блоком
   «почему мы» с цифрами; учесть локально-справочный слой выдачи (карточка, отзывы, адрес).
3. **Цена / процесс / FAQ.** Добавить блок-ответ «Сколько стоит разработка ПО на заказ» (закрывает
   поддерживающий спрос 253 и вопросный «сколько стоит») с перелинковкой на `/stoimost/`; FAQ с
   вопросами про цену, сроки, права на код, модель оплаты (интенты подтверждены выдачей и статьями).
4. **Подразделы и кейсы.** Блок «Когда нужна заказная разработка» (закрывает информационный слой из
   3 статей топа); блок «Что разрабатываем» (раскрыть скрытый каталог — перелинковка на `/uslugi/*`);
   кейсы с цифрами (доверие). Отзывы/логотипы — раскрыть скрытый блок (фактор «компания + отзывы»).
5. **Перелинковка.** Цена → `/stoimost/`; сроки → `/sroki/`; гарантии → `/garantii/`; конкретные
   услуги → `/uslugi/*`; импортозамещение → `/importozameshchenie/`. Анкоры — по смыслу.
6. **Чего не делать.** Не таргетировать голое «разработка программного обеспечения» (образовательный
   интент). Не дублировать ценовой макрокластер с `/stoimost/` (главная — только поддерживающий блок).
   Не добавлять списки запросов и SEO-простыни; «под ключ» — максимум один раз естественно.

## План мониторинга

- **Базовый период:** 15.06–15.07.2026 — по целевому коммерческому кластеру показов/кликов у URL
  нет (только бренд). Это стартовая точка «с нуля».
- **После внедрения** (и переобхода): сравнивать сопоставимые периоды по показам URL, кликам, CTR,
  средней позиции и числу релевантных **небрендовых** запросов главной; отдельно следить, не
  «перетягивает» ли главная запросы услуг у `/uslugi/*` (риск каннибализации из-за общего hero).
- **Условия пересмотра:** изменение предложения/структуры сайта, заметный сдвиг выдачи или спроса,
  либо плановый пересмотр через 3–6 месяцев. Для актуализации частотностей — при следующем проходе
  собрать «Подбор запросов и анализ рынка» в Вебмастере (в этот раз инструмент был недоступен).
