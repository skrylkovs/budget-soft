## Рекомендации по странице

1. **Title / H1 / Description:** Title — «Заказать UX/UI-дизайн интерфейсов — [цена от] ₽: дизайн сайтов, мобильных приложений и систем»; H1 — «UX/UI-дизайн интерфейсов на заказ»; Description — цельными предложениями: что дизайним (веб/мобильные/корпоративные), прототип за 2 недели, передача в разработку без потерь. P0 в начале, без перечней ключей.
2. **Первый экран:** сразу ясно — это заказ дизайна (не курсы): «проектируем интерфейсы… передаём в разработку»; порог входа и срок прототипа.
3. **Цена / процесс / FAQ:** конкуренты либо прячут цены, либо дают тарифы (ddsi 10–35 тыс., persik 30–150 тыс., thelightech от 1 600 ₽/час) — дать пакеты с цифрами (все новые цифры `[≈ согласовать]`) и ставку из аутстаффинга (1 600 ₽/час — факт сайта). FAQ: сколько стоит, сроки (4 нед. медиана рынка), что получу, UX vs UI, редизайн, передадите ли в разработку.
4. **Подразделы:** мобильный дизайн (подкластер 2), веб/корпоративные интерфейсы (подкластер 3 — ниша bpadevelop), прототип за 2 недели (подкластер 4 + факт сайта), дизайн-системы/UI-kit (подкластер 5 — пустая ниша), этапы с артефактами (паттерн purrweb).
5. **Перелинковка:** входящие — с mobilnaya-razrabotka, razrabotka-sajtov, razrabotka-mvp, it-autstaffing (роль дизайнера); исходящие — ux-audit, bi-big-data, telegram-mini-apps, razrabotka-sajtov по таблице передачи.
6. **Чего не делать:** не таргетировать «ux ui дизайн» голым заголовком (образовательная выдача), не вписывать города/списки запросов, не выдумывать кейсы дизайна, которых нет в портфолио.
