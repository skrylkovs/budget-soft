## Методика сбора

- **Маркерные формулировки:** `разработка маркетплейса`, `создание маркетплейса`, `маркетплейс под ключ`.
- **Проверенные маски (Вордстат, Россия):** `разработка маркетплейса` (база 650), `создание маркетплейса` (13 368 — ~85% шум «карточек для маркетплейсов»), `маркетплейс под ключ` (297 — преимущественно селлерский шум), `b2b маркетплейс` (703 — инф./навигационный), `стоимость маркетплейса` (1 315 — селлерский шум); плюс срезы Подбора запросов Вебмастера по двум маркерам.
- **Критерий остановки расширения:** маски «стоимость», «b2b», «под ключ» новых релевантных подкластеров не открыли — только шум услуг для продавцов на WB/Ozon; дальнейшие комбинации повторяют собранные интенты.
- **Нормализация:** исходные формулировки сохранены как в источниках; дедупликация по лемме без изменения порядка слов; ё/е не различаются.
- **Интент:** проверен по органическому топ-10 Яндекса (lr=213, 06.07.2026, desktop) для 4 формулировок; реклама (`yabs.yandex.ru`, `yandex.ru/an/count`) и колдунщики (Алиса, видео) исключены.
- **SERP-кластеризация:** формальный подсчёт пересечений не применялся (ручная кластеризация + автокластеры Подбора). Автокластеризация Яндекса относит «создание маркетплейса под ключ/цена/москва» к кластеру «разработка маркетплейса» — интенты совместимы, один URL. Пересечение топов «разработка…» и «…под ключ» сильное (cs-cart, uslugi.yandex, surf, o2k в обоих).
- **Ограничения:** собственной статистики URL нет (не в индексе под www); «спрос» Подбора (среднемес. за год) и Вордстат (30 дней) — разные метрики, в таблицах не смешиваются; ниша низкочастотная — месячные значения точных форм статистически шумные.

- **Базовый период:** данных нет — URL отсутствует в статистике Вебмастера (хост-блокер). База = ноль показов до индексации под www.
