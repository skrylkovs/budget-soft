# SEO-семантика: Интеграция с ЕСИА — вход через Госуслуги

## Scope

- **URL:** `/uslugi/integraciya-s-esia/` — **новая** страница (создаётся в этом цикле).
- **Дата сбора:** 20.07.2026.
- **Slug в SERVICE_META:** `integraciya-s-esia`, позиция **97**.
- **Задача ядра:** свести в один URL два разных языка одного спроса —
  разговорный («вход/авторизация через Госуслуги») и документарный («ЕСИА»,
  «подключение к ЕСИА»), не отдав ни одну из групп смежным страницам.

> «частотность не снята», — экспертная гипотеза по составу выдачи, а не
> измеренное число. Ядро подлежит доснятию при свободном браузере.

Соотношение 5825 : 816 — разница почти в 7 раз, но **вытеснять «ЕСИА» нельзя**:
это язык тендерного ТЗ и технического задания, то есть язык того, кто
подписывает договор. Разговорная группа приводит трафик, документарная —
сделку. Обе обязаны быть в Title.

## Краткий вывод

1. **Вход в спрос — разговорный, конверсия — документарная.** Посетитель ищет
   «авторизацию через Госуслуги», а в ТЗ у него написано «интеграция с ЕСИА».
   Страница обязана говорить обоими языками и явно связывать термины между
   собой: «вход через Госуслуги» = технически «интеграция с ЕСИА».
2. **Рынок делится надвое, и это главная развилка страницы.** Коробочные
   модули под Битрикс (marketplace.1c-bitrix.ru, unicoms, alfamax, bitrix.g-i-t)
   против интеграции под ключ в собственную систему (esia.ru, web-canape,
   cetera, esia.nan.ru, mcart) и SaaS-проксирования (esia.cloud, TrustGate от
   «Инфотекс»). **Наша ниша — вторая.**
3. **Цену не публикует НИКТО** — ни модульщики, ни интеграторы, ни SaaS.
   Публикация вилки — сильнейший дифференциатор этой ниши, сильнее любого
   текстового преимущества.
4. **Пустая ниша №1 — честная граница ответственности.** Подключение к ЕСИА
   регламентная процедура: заявка через оператора, работа в тестовой среде,
   требования по защите информации, согласования. Решение принимает оператор,
   а не подрядчик. Конкуренты этого не проговаривают — страница, которая
   объяснит границу прямо, выигрывает доверие тендерного читателя.
5. **Пустая ниша №2 — разведение «модуль или интеграция» до сметы.** Заказчику
   на Битриксе не нужен проект за 120 000 ₽; заказчику со своей системой модуль
   не поможет вовсе. Развилка, отданная честно, экономит обеим сторонам цикл.
