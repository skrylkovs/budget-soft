### Критические ограничения доступа и индексации (проверено 06.07.2026 — без изменений)

1. В Вебмастере подтверждён только хост `https://budget-soft.ru` (не-www). Фактический
   канонический хост `https://www.budget-soft.ru` в Вебмастер **не добавлен** — статистика
   www-хоста недоступна.
2. Не-www URL услуг отдают **301-редирект** на www (история обхода 24.06.2026: `N/a → 301`
   по страницам `/uslugi/*`). Под каноническим www-адресом целевая страница не отслеживается.
3. В поиске Яндекса находится **только главная** (обновление 28.06–05.07.2026: 0 добавлено /
   0 удалено; ИКС — «недостаточно данных»). По сайту виден один бренд-запрос
   `budget-soft.ru` → `/` (3 показа / 1 клик). Нулевая видимость целевого URL — следствие
   возраста сайта и незавершённого переезда на www, а не качества страницы.

  программистов, аренда программиста).
- **Критерий остановки расширения:** «Похожие запросы» Вордстата по `it аутстаффинг` дали
  нерелевантный шум («айтишник», «айти сфера» и т. п.); маска «аренда программиста» замкнулась

  модель «аутстаффинг {стек} разработчиков». Новые релевантные интенты перестали появляться —
  смысловое насыщение достигнуто.
- **Нормализация:** исходные формулировки сохранены; дубли сведены по нижнему регистру,
  ё/е, латиница/кириллица (`it`/`ит`/«айти»); перестановки и словоформы не склеивались без
  операторной проверки.
- **Интент:** топ-10 органики Яндекса, Москва (lr=213), desktop, 06.07.2026 — по 4 запросам
  (см. ниже). Рекламные блоки (`yandex.ru/an/count`) и агрегаторы не считались органическим
  URL целевого типа.
- **SERP-кластеризация:** формальная (top-N/порог) не использовалась — решения приняты
  вручную по типу документов в выдаче; спорные строки помечены «требуется проверка».
- **Ограничения:** нет статистики URL (страница не в индексе под www); «спрос» Подбора и

## Рекомендации по странице

1. **Title / H1 / Description:** H1 должен называть услугу словами P0 — сейчас H1 hero
   обобщённый («Разрабатываем ПО любой сложности…»), тема живёт только в H2. Рекомендуемый
   каркас H1: «IT-аутстаффинг: разработчики и IT-специалисты Middle+/Senior в вашу команду».
   Title — «it аутстаффинг» ближе к началу + факты (уровень, срок вывода от 3 дней);
   Description — цельные предложения с фактами (T&M, от 3 рабочих дней, роли), уникальный
   в пределах сайта. Не обещать дословный показ в сниппете.
2. **Первый экран:** сразу после H1 — лид с ценностным предложением (кого даём, как быстро,
   по какой модели оплаты) + spec-карточки Цена/Сроки/Формат (уже есть — сохранить).
3. **Цена:** добавить H2 «Сколько стоит IT-аутстаффинг» с текстовым ответом в 2–3 предложения
   и рейт-картой по грейдам/ролям (ставки согласовать с бизнесом — на сайте ставок нет);
   перечислить факторы ставки и модель T&M.
4. **Роли и стек:** блок направлений (backend, frontend, мобильная, QA, DevOps, аналитика,
   1С) с технологиями внутри карточек — закрывает подкластеры 2–4 без переспама.
5. **Процесс:** этапы вывода специалиста с длительностями (заявка → подбор кандидатов →
   интервью → выход в команду от 3 рабочих дней) и что берёт на себя провайдер (зарплата,
   налоги, оборудование, замена).
6. **Сравнение:** переработать существующий список «аутсорсинг vs аутстаффинг» в таблицу;
   из неё — перелинковка на `/uslugi/it-autsorsing/`.
7. **FAQ:** 8–12 вопросов — «что это», «сколько стоит/от чего зависит ставка», «как быстро»,
   «чем отличается от аутсорсинга», «кто управляет специалистом», «как оформляется договор/
   НДА», «можно ли заменить специалиста», «нужен специалист в штат» (→ it-rekruting).
   Синхронизировать с FAQPage-разметкой 1:1.
8. **Перелинковка:** по таблице передачи (it-autsorsing, it-rekruting, devops-uslugi);
   входящие ссылки — с `/uslugi/` и профильных страниц.
9. **Чего не делать:** не выводить города списком; не создавать H2 под каждый стек; не
   выдумывать ставки/кейсы; не таргетировать «аутсорсинг» и рейтинг-запросы; не превращать
   FAQ в перечень запросов.
