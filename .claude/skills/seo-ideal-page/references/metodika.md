# Методика: сбор выдачи, разбор конкурентов, синтез и копирайт

## 1. Сбор органической выдачи Яндекса

### Прямой переход на поисковый URL

Используй Playwright MCP (`browser_navigate`), не форму поиска:

```
https://yandex.ru/search/?text=<url-encoded запрос>&lr=<регион>
```

- `lr=213` — Москва (по умолчанию для федерального продвижения этого проекта).
- Одна навигация = один запрос. Между запросами ничего ждать не нужно.
- Заголовок страницы вида «… — Яндекс: нашлось N результатов» подтверждает,
  что выдача отдана, а не капча.

### Извлечение результатов

Сразу после навигации выполни `browser_evaluate`:

```js
() => {
  const results = [];
  document.querySelectorAll('li.serp-item').forEach(li => {
    const a = li.querySelector('a.Link[href^="http"], a.organic__url, a[href^="http"]');
    const titleEl = li.querySelector('h2, .OrganicTitle, .organic__title');
    const snippetEl = li.querySelector('.OrganicTextContentSpan, .organic__content-wrapper, .TextContainer');
    if (a && titleEl && !a.href.includes('yabs.yandex.ru')) {
      results.push({
        url: a.href.slice(0, 160),
        title: (titleEl.textContent || '').trim().slice(0, 130),
        snippet: snippetEl ? (snippetEl.textContent || '').trim().slice(0, 200) : ''
      });
    }
  });
  return JSON.stringify(results, null, 1);
}
```

### Классификация результатов

| Признак | Класс | Что делать |
|---|---|---|
| `yabs.yandex.ru`, `yandex.ru/an/count` в href | реклама | исключить из разбора; сниппеты можно читать как источник офферов рынка |
| profi.ru, uslugi.yandex.ru, ya.ru/services, маркетплейсы услуг | агрегатор | не разбирать постранично; зафиксировать сам факт (работает фактор «цена + отзывы») |
| habr.com, medium.com, vc.ru, блоги компаний, СМИ | статья | разобрать отдельно как источник контентных идей; сигнал, что Яндекс подмешивает объясняющий контент |
| страница услуги компании-разработчика | **конкурент** | полный разбор |
| иноязычные страницы | шум | исключить |

Итог: 8–15 уникальных конкурентных URL. Для каждого пометь, по каким запросам он
встретился (встречается в 2+ запросах = сильный конкурент, разбирать первым).

### Протокол капчи

Если вместо выдачи открылась капча («Подтвердите, что вы не робот», SmartCaptcha):
не решай её сам и не обходь. Скажи пользователю, что в открытом окне браузера нужно
пройти проверку, дождись подтверждения и повтори навигацию. Если пользователь
недоступен — зафиксируй ограничение и работай с тем, что успел собрать.

## 2. Разбор страницы конкурента

### Базовый экстрактор

`browser_navigate` на URL конкурента, затем `browser_evaluate`:

```js
() => {
  const clean = s => (s || '').replace(/\s+/g, ' ').trim();
  const meta = document.querySelector('meta[name="description"]');
  const headings = [...document.querySelectorAll('h1,h2,h3')]
    .map(h => h.tagName + ': ' + clean(h.textContent).slice(0, 120))
    .filter(t => t.length > 4).slice(0, 50);
  const body = document.body.innerText || '';
  const priceLines = [...new Set(body.split('\n').map(clean)
    .filter(l => /₽|руб\.|стоимост|цена/i.test(l) && l.length > 10 && l.length < 220))].slice(0, 15);
  const timeLines = [...new Set(body.split('\n').map(clean)
    .filter(l => /(месяц|недел|срок)/i.test(l) && l.length > 10 && l.length < 180))].slice(0, 10);
  const faq = [...new Set(body.split('\n').map(clean)
    .filter(l => l.endsWith('?') && l.length > 15 && l.length < 150))].slice(0, 15);
  const ctas = [...new Set([...document.querySelectorAll('button, a[class*="btn"], a[class*="button"]')]
    .map(b => clean(b.textContent)).filter(t => t.length > 3 && t.length < 60))].slice(0, 12);
  return JSON.stringify({ title: document.title, desc: meta ? meta.content : '',
    headings, priceLines, timeLines, faq, ctas, chars: body.length }, null, 0);
}
```

### Точечное дочитывание

Если в заголовках есть ценовой блок («Стоимость…», «Тарифы», «Пакеты»), а в
`priceLines` нет цифр — цифры отрисованы отдельными узлами. Вытащи секцию целиком:

```js
() => {
  const body = document.body.innerText || '';
  const i = body.indexOf('Стоимость');           // или точный заголовок блока
  return body.slice(i, i + 1800).replace(/\n{2,}/g, '\n');
}
```

### Что фиксировать по каждому конкуренту

- Title, Description, H1 — как сформулирован главный интент; есть ли цена в Title.
- Порядок смысловых блоков (по H2/H3) — это карта их страницы.
- **Цены:** конкретные цифры, пакеты (состав/цена/срок), калькулятор, «цена не от»,
  либо честное «цен нет».
- **Сроки:** полный цикл, MVP, гарантийный период.
- **Доверие:** кейсы с цифрами эффекта, количество проектов, годы, отзывы, награды,
  NDA, гарантии, SLA.
- **FAQ:** формулировки вопросов дословно (это подтверждённые интенты).
- **CTA:** что предлагают за контакт (аудит, концепция, расчёт, демо) — чем ниже
  порог, тем сильнее.
- **Слабости:** нет цен, нет FAQ, тонкий текст (< 5–7 тыс. знаков), два H1,
  обещание в Title без ответа на странице, каша в структуре.

### Замеченные маркеры качества страницы (что «нравится» топу)

Наблюдения из фактических разборов (перепроверяй на своей нише):

- пакеты цен таблицей «состав → цена → срок» — стандарт сильных страниц;
- пары «проблема → что меняется» с деньгами в формулировках — самый продающий блок;
- вопросные H2 словами спроса («Сколько стоит…», «Что выбрать…») — статейный слой
  внутри коммерческой страницы;
- сравнение «коробка vs заказ» таблицей;
- цена прямо в Title — выделяет сниппет, когда у большинства цен нет;
- FAQ, совпадающий с чужим дословно, — антимаркер (фиксируй как проблему).

## 3. Синтез

1. **Гигиенический минимум** = блоки, которые есть у ≥ 2/3 сильных конкурентов
   (H1 с точным интентом, цена «от», этапы, кейсы с цифрами, стек, интеграции, FAQ,
   CTA, цифры компании). Сравни с нашей страницей: есть / нет / слабо.
2. **Лучшие находки** — топ-5..10 конкретных ходов с указанием источника
   («у кого подсмотрено»). Ход = что сделать + почему работает.
3. **Слабости и пустые ниши** — то, чего не делает никто или почти никто:
   незакрытый ценовой интент, отсутствие быстрого старта, пустые сервисные
   формулировки. Каждую слабость превращай в наш блок или тезис.
4. **Ценовое позиционирование** — собери фактическую вилку рынка из извлечённых цен
   и определи, где мы (ниже порога всех / в середине / выше). Это определяет,
   насколько агрессивно показывать цену.
5. Все обобщения («у всех», «почти никто») должны быть арифметически верны по
   таблице разбора; для части — пиши «у N из M».

## 4. Правила продающего копирайта

- **Каждый блок = интент.** Не пиши блок, если он не отвечает на запрос ядра, не
  закрывает слабость конкурентов и не снимает возражение покупателя.
- **Лид первого экрана:** P0-интент + оффер + строка фактов с цифрами
  (цена «от», срок MVP, проекты, годы). Сервисная формулировка «услуги по…» —
  если ядро подтверждает такой спрос.
- **Ценовой блок обязателен**, если ценовые запросы в P0/P1: абзац-ответ в 2–3
  предложения сразу под H2 (годится для сниппета), таблица пакетов
  «состав → цена → срок», 3–5 факторов цены, модели оплаты. Реальные цифры — с
  сайта; предлагаемые — `[≈ согласовать]`.
- **Пары «проблема → решение»:** 4–6 пар, в «проблеме» — узнаваемая боль с
  конкретикой (где лежат данные, что считается вручную), в «решении» — измеримое
  изменение. Без абстракций «повысим эффективность».
- **Вопросные H2/H3** используй для подтверждённого вопросного спроса; ответ — в
  первом же абзаце после заголовка.
- **Этапы** — с длительностями и проверяемым результатом каждого этапа.
- **Перелинковка** — по таблице «Передать другим URL» из ядра + профильные услуги
  из блоков (модули, кейсы); анкор по смыслу, без «подробнее тут».
- **FAQ 8–12 вопросов:** сначала деньги и сроки (вопросный спрос ядра дословно по
  интенту), затем процесс, права, поддержка. Ответы — только реальные факты.
- **Тон:** конкретно, с цифрами, без превосходных степеней и «лидеров рынка».
  Сравнения с конкурентами — обезличенно («компании из топа выдачи»), без имён
  в копирайте (имена — в приложении А).
- **Транслит/варианты** («ерп») — максимум один раз, естественно, в скобках.

## 5. Ограничения и честность

- Числа из разных страниц и периодов не суммируй и не усредняй без пометки.
- Извлечённый `innerText` может не содержать лениво-подгружаемых блоков: если
  страница выглядит подозрительно «тонкой», отметь это, а не записывай в слабости
  безоговорочно.
- Выдача персонализируется и меняется: фиксируй дату и регион сбора в документе.
- Субагенты для параллельного разбора — опционально; при ошибках лимитов не
  повторяй попытки, переходи на последовательный браузерный разбор.
