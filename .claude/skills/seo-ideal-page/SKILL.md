# Идеальная страница по конкурентам из топа Яндекса

Изучи фактический органический топ Яндекса по целевым запросам страницы, разбери
страницы конкурентов и собери контент-документ «идеальной страницы»:
готовый продающий копирайт + обоснование каждого решения данными конкурентов и ядра.
Результат — `seo-pages/<путь-страницы>.md`. Саму страницу этот скилл **не меняет**.

Перед работой прочитай:

- [references/metodika.md](references/metodika.md) — сбор выдачи, экстрактор страниц
  конкурентов, правила фильтрации, синтеза и продающего копирайта;
- [references/shablon.md](references/shablon.md) — обязательный формат результата.


Место в конвейере: `seo-semantic-core` собирает ядро (`seo-core/<путь>.md`) →
**этот скилл** строит эталонный контент по конкурентам (`seo-pages/<путь>.md`) →
`seo-page-optimization` внедряет решения на страницу и создаёт отчёт
(`seo-optimization/<путь>.md`).

## Входные данные

1. **URL или путь страницы.** Проверь наличие пути в [urls.json](../../../urls.json).
   Если страница не указана и её нельзя однозначно определить — спроси.
2. **Семантическое ядро** `seo-core/<путь>.md` — желательно, но не обязательно.
   Если ядро есть: возьми из него P0/P1/P2-запросы, подкластеры, SERP-выводы и
   рекомендации — они управляют выбором поисковых формулировок и структурой документа.
   Если ядра нет: не собирай его на лету — возьми 2–4 маркерные формулировки из
   Title/H1 страницы, зафиксируй в документе, что ядро отсутствует, и предложи
   выполнить seo-semantic-core отдельно.
3. **Регион выдачи.** По умолчанию Москва (`lr=213`) для федерального продвижения
   этого проекта; если в ядре или у пользователя другой регион — используй его.
4. **Инвентарь фактов компании.** До написания копирайта собери с текущей страницы и
   соседних страниц сайта все реальные факты: цены, сроки, этапы, стек, кейсы с
   цифрами, гарантии, модели оплаты, контакты. Это единственный источник фактов
   о компании.

## Неподменяемые правила

- **Страницу и её URL не менять.** Результат скилла — только документ
  `seo-pages/<путь>.md`. Правки страницы — отдельная задача (seo-page-optimization).
- **Факты компании — только с сайта или от пользователя.** Каждая цифра в копирайте
  (цена, срок, кейс, гарантия) обязана иметь источник на сайте. Предлагаемые новые
  цифры (например, промежуточные ценовые вилки) помечай маркером `[≈ согласовать]`
  и перечисляй в чек-листе внедрения.
- **Данные конкурентов — только из фактически полученных страниц.** Не выдумывай
  цены и блоки «по памяти». Если страница не отдалась — пиши `н/д` и причину.
  Гипотезы, не подтверждённые парсингом, помечай словом «гипотеза».
- **Не копируй текст конкурентов.** Перенимай паттерны (тип блока, способ подачи цены,
  формулировку вопроса FAQ как интент) — формулировки пиши свои. Прямое заимствование
  фраз запрещено; выявленные совпадения текущей страницы с конкурентами фиксируй как
  проблему.
- **Рекламу и агрегаторы отделяй от органики.** Ссылки `yabs.yandex.ru` и
  `yandex.ru/an/count` — реклама; profi.ru, uslugi.yandex.ru, ya.ru/services и
  карты — агрегаторы. В таблицу конкурентов для разбора идут коммерческие страницы
  услуг из органики; статьи из топа разбирай отдельно как источник контентных идей.
- **Структуру документа собирай с нуля — её диктуют конкуренты и ядро, а не
  страницы сайта.** Порядок блоков сверху вниз выбирай по логике покупателя и
  данным анализа (что топ считает гигиеническим минимумом, где пустые ниши):
  структура документа станет структурой страницы. Не собирай документ по
  составу секций существующих страниц сайта (ERP/CRM и др.) — они дают только
  типовые механизмы оформления. Каждая строка «Лучших находок» и «пустых ниш»
  приложения А обязана либо стать блоком/элементом копирайта, либо получить
  явный отказ с причиной прямо в документе. Документ, состав которого зеркалит
  существующую страницу сайта, — не результат этого скилла.
- **Анти-переспам.** Копирайт пишется для человека: никаких перечней запросов,
  «ключей ради ключей» и городов списком. Покрытие ядра проверяй по смысловому
  ответу на интент, леммы — вторичная проверка.
- **Объём — по конкурентам, вода — под нож.** Не делай страницу большой ради
  объёма. Целевой размер контента — диапазон сильных страниц топа (ориентир —
  медиана, не максимум); если топ короткий — страница короткая. Каждое предложение
  несёт факт, цифру или конкретику; при выборе между «добрать объём» и «убрать
  воду» — всегда убирай воду, даже ценой меньшего объёма.
- **Не обещай позиций.** Документ обосновывает решения данными, но не гарантирует
  ранжирование.

## Рабочий процесс

### 1. Подготовка

1. Найди путь в `urls.json`, прочитай текущую страницу целиком (`<путь>/index.html`).
2. Прочитай ядро `seo-core/<путь>.md`, если есть: выпиши P0/P1/P2 по подкластерам,
   главные контентные пробелы, таблицу «Передать другим URL» (для перелинковки)
   и уже готовые SERP-выводы.
3. Составь инвентарь фактов компании (см. «Входные данные», п. 4) и список страниц
   сайта для перелинковки (по `urls.json`).
4. Выбери 2–4 поисковые формулировки: главный P0-запрос, вариант «на заказ»/«под
   ключ», смежный подкластер (например «внедрение …»). Не трать формулировки на
   запросы, которые ядро пометило «передать другим URL» или «исключено».

### 2. Сбор выдачи Яндекса

Собери органический топ по каждой формулировке через браузер (Playwright MCP),
как описано в методике: прямой переход на поисковый URL с `lr`, извлечение
результатов JS-сниппетом, фильтрация рекламы и агрегаторов. Капча — по протоколу
из методики (решает пользователь в открытом окне). Итог шага — список из
8–15 уникальных конкурентных URL с пометками (тип: услуга/статья/агрегатор,
в каких запросах встретился, что обещает сниппет).

### 3. Разбор страниц конкурентов

Для каждого конкурентного URL: открой страницу в браузере и извлеки экстрактором из
методики Title, Description, H1–H3 по порядку, строки с ценами, строки со сроками,
вопросы FAQ, CTA и объём текста. Для страниц с ценовыми блоками сделай
дополнительное точечное извлечение секции с цифрами. Статьи из топа разбирай на
уровне оглавления и подачи цены. Фиксируй неудачи честно (`н/д` + причина).

Параллельный разбор субагентами допустим, но необязателен: извлечение через браузер
в основном цикле надёжнее и не тратит лимиты; при падении субагентов (лимит сессии,
таймауты) переходи на браузерный разбор, а не жди.

### 4. Синтез

Сведи разбор в четыре списка (подробные критерии — в методике):

1. **Гигиенический минимум** — что есть у всех сильных страниц топа; отметь, что из
   этого уже есть у нас, чего нет.
2. **Лучшие находки** — конкретные контентные ходы с указанием, у кого подсмотрено.
3. **Слабости конкурентов** — где ниша пуста и на чём выигрывать (например: никто
   не отвечает «сколько стоит», у всех порог цены выше нашего, ни у кого нет
   быстрого MVP).
4. **Выводы по составу выдачи** — доля коммерческих страниц, наличие статей
   (сигнал: нужен объясняющий текстовый слой), агрегаторов (сигнал: работает
   фактор «цена + отзывы»).
5. **Ориентир по объёму** — медиана объёма текста сильных конкурентов (поле
   `chars`, как относительный сигнал, выбросы отбрось). Задаёт целевой размер
   страницы: около медианы, а не по максимуму. Если топ короткий — страница
   короткая.

### 5. Сборка документа

Собери `seo-pages/<путь>.md` строго по [references/shablon.md](references/shablon.md):
мета-данные (Title/Description/H1 с обоснованием), готовый копирайт всех блоков
сверху вниз, FAQ, приложения А/Б/В (конкуренты, покрытие ядра, чек-лист внедрения).
Правила продающего копирайта — в методике; главное: каждый блок отвечает на
конкретный интент ядра или закрывает выявленную слабость конкурентов, и это
обоснование указано в документе. Объём калибруй по конкурентам (около медианы
топа, не по максимуму) и режь воду: каждое предложение несёт факт или конкретику;
если топ отвечает коротко — отвечай коротко.

### 6. Верификация

1. **Фактология:** пройди по каждой цифре копирайта — источник на сайте либо маркер
   `[≈ согласовать]`. Цифры конкурентов сверь со своими извлечениями.
2. **Покрытие:** каждая строка P0/P1 ядра имеет назначенный блок в документе или
   явное объяснение, почему не таргетируется.
3. **Анти-переспам-проход:** прочитай копирайт как редактор; ни одна лемма не
   доминирует, формулировки естественны.
4. **Объём и вода:** сравни объём копирайта с медианой топа — не раздут ли он
   сверх диапазона конкурентов без явной причины. Пройди на воду: каждое
   предложение несёт факт/цифру/конкретику; предложения-пустышки вырежи, даже
   если страница станет короче.
5. **Правдивость обобщений:** утверждения вида «у конкурентов X» должны совпадать
   с таблицей приложения А (не обобщай на всех то, что верно для части — смягчай
   формулировку).

### 7. Отчёт пользователю

Сообщи: путь к документу; сколько страниц конкурентов разобрано и какие не отдались;
3–5 главных выводов анализа (пустые ниши, наше преимущество); какие цифры ждут
согласования (`[≈ согласовать]`); напомни, что внедрение на страницу — следующий шаг
(seo-page-optimization), и какие блокеры индексации известны из ядра.

## Чек-лист качества

- [ ] Выдача собрана живьём из Яндекса (не по памяти), регион зафиксирован.
- [ ] Реклама и агрегаторы отфильтрованы; разобрано ≥ 6 конкурентных страниц
  (либо объяснено, почему меньше).
- [ ] Все цифры конкурентов взяты из фактических извлечений; неполученное — `н/д`.
- [ ] Каждый факт о компании имеет источник на сайте; новые цифры — `[≈ согласовать]`.
- [ ] Текст конкурентов не скопирован; совпадения текущей страницы с конкурентами
  зафиксированы.
- [ ] Каждая строка P0/P1 ядра покрыта блоком или объяснением.
- [ ] Каждая «лучшая находка» и «пустая ниша» приложения А реализована блоком
  или явно отклонена с причиной; состав документа не зеркалит существующие
  страницы сайта.
- [ ] Запросы «передать другим URL» не таргетируются — только перелинковка.
- [ ] Анти-переспам-проход выполнен; списков городов и перечней запросов нет.
- [ ] Объём откалиброван по топу (около медианы, не максимум); воды нет — каждое
  предложение несёт факт/цифру/конкретику, страница не раздута.
- [ ] Документ соответствует шаблону: мета, копирайт, FAQ, приложения А/Б/В.
- [ ] Сама страница и `urls.json` не изменены.
- [ ] Блокеры из ядра (Вебмастер, индексация) продублированы в чек-листе внедрения.
