## Требования к результату

- Не смешивать в одной числовой колонке Вордстат, спрос Вебмастера и показы URL.
- Не считать отчёт завершённым без авторизованной проверки правильного подтверждённого
  сайта в Яндекс Вебмастере.
- У каждого числа должны быть источник, период и настройки.
- Внутри подкластеров сначала располагать `P0`, затем `P1`, `P2`; при равном приоритете
  сортировать по сопоставимому спросу.
- Один запрос не дублировать между целевыми подкластерами.
- Запрос другого URL переносить в соответствующую таблицу, а не оставлять в ядре.
- Информационный запрос маршрутизировать по интенту, а не исключать автоматически.
- Исходные формулировки сохранять; нормализацию использовать только для анализа дублей.
- Комментарий должен объяснять решение, а не повторять значение соседних колонок.
