### Наблюдения по выдаче и ценам

Разобрано 7 источников (worksolutions, evrone, scand, sibdev, surf, programmingtech, profi-цены); axmor.ru/java-development дал 404. Ключевые наблюдения: (1) ЦЕНЫ почти никто не публикует за проект — evrone от 2 800 ₽/час, profi/фриланс от 1 400–1 600 ₽/час (Java/Spring); проектных цен «от … ₽» в теле страниц нет ни у одного → ценовая ниша свободна, показываем матрицу с ценами. (2) ТИПЫ проектов на Java у конкурентов — enterprise/финтех/банкинг (worksolutions: FinTech, ERP/CRM, HRTech, PropTech; scand: Enterprise ERP/CRM/HRM, Fintech, Big Data, Blockchain), highload-бэкенд (surf: KFC 900+ филиалов, Delivery Club, Lamoda), микросервисы, интеграции/миграции legacy (evrone, scand), billing/WMS/MES (programmingtech). Почти никто НЕ продаёт «сайты/лендинги» на Java — стек читается как enterprise-бэкенд; отсюда матрица у нас enterprise/highload, а не «сайты/магазины». (3) СРАВНЕНИЕ: scand прямо перечисляет «.Net vs Java / PHP vs Java / Python vs Java / Kotlin vs Java», programmingtech — «Java быстрее JS/PHP/Python» → отсюда FAQ «Java или .NET/PHP» и таблица Java(Spring) vs PHP/Node.js. (4) FAQ конкурентов: стоимость, сроки, что разрабатывают, доработка чужого кода (evrone, worksolutions), поддержка после релиза, интеграция с существующими системами, аутсорс vs аутстафф (worksolutions), «какой язык выбрать» (scand). (5) ЭКОСИСТЕМА (сквозная у всех): Spring Boot/Spring Cloud, Hibernate/JPA, Kafka/RabbitMQ, Kubernetes/Docker, PostgreSQL/Redis/Elasticsearch, Keycloak, JUnit/TestNG/Testcontainers, gRPC, Prometheus/Grafana. (6) БОЛИ: масштабирование под рост нагрузки, legacy/монолит без документации, недостаток внутренней экспертизы, миграция на свою команду, контроль времени разработчиков. Медиана объёма сильных конкурентов — компактная (evrone, worksolutions), простыни — у scand/programmingtech; целевой объём 7–9 тыс. знаков соблюдён.

## Приложение Б. Каннибализация и отклонения от каркаса

Локальные отклонения от эталона Laravel (обоснованы выдачей: Java в топе Яндекса/поиска читается как enterprise/highload-бэкенд, а не «сайты/магазины»): (1) Матрица «Что строим» — вместо строк «корпоративный сайт/магазин» использованы enterprise/highload-строки (личный кабинет/B2B, ERP, веб-приложение+API, микросервисы, highload-бэкенд, финтех); каждая ведёт на реальную страницу /uslugi/, порядок блоков и маркеры каркаса сохранены 1:1. (2) Таблица сравнения — не «Java vs .NET» (это стек одного веса, ставить .NET в проигрышную колонку некорректно), а «Java (Spring) vs PHP/Node.js» — граница, где Java избыточна; прямой выбор Java/.NET/PHP вынесен в FAQ №3 по требованию задачи. (3) Порог входа выше, чем у Laravel (250 000 ₽ против 150 000 ₽): Java — более тяжёлый стек, «сайт-визитка от 150к» в матрицу не ставилась осознанно. Риск каннибализации со страницами /uslugi/ (razrabotka-veb-prilozhenij, razrabotka-mikroservisov, vysokonagruzhennye-sistemy, fintech, razrabotka-erp-sistem) низкий: лендинг ведёт трафик НА них, интент иной — «сделать это на Java» против «что это за услуга»; H1 и Title таргетируют стек, а не тип решения.
