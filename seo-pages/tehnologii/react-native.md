### Наблюдения по выдаче и ценам

Выдача (US-класс. недоступен, работал по органике поиска + разбор 8 страниц студий РФ/СНГ). H1 у всех содержит «React Native», формулировки — «разработка на React Native», «разработка мобильного приложения на React Native», «React Native приложения на заказ». Главная альтернатива для сравнения — НАТИВНАЯ разработка (iMedia, Purrweb, LighTech прямо противопоставляют; RN не тянет игры/3D/AR/VR), вторичная — Flutter (Purrweb называет ближайшей альтернативой, Evrone — статья «Flutter vs React Native»). Универсальный оффер конкурентов: единая кодовая база = экономия 30–40% против двух нативных команд + быстрый MVP для стартапов. Цены: разброс от 600К (DDSI база) до 1,5 млн+ (DDSI топ), LighTech проекты от 1 млн / от 2500 ₽/час; общерыночная вилка простое 1,5–3 млн, среднее 3–7 млн, сложное от 8 млн ₽. Сроки: MVP 2–3 мес, среднее 4–6 мес, сложное 6–12 мес. Ниши слабо закрыты конкурентами: двухмодельный ценовой блок (Fixed Price + T&M), доработка чужого RN-кода/обновление устаревших версий, аутстафф именно RN-инженеров, OTA-обновления как преимущество. Экосистемные инструменты в выдаче: TypeScript, React Navigation, Reanimated, Redux/Zustand/Recoil, Clean Architecture, Firebase, Expo, нативные модули, публикация RuStore/Google Play/App Store.

## Приложение Б. Каннибализация и отклонения от каркаса

Каркас эталона сохранён 1:1, локальных отклонений от структуры нет: матрица «Что строим» (6 строк) → {verdict-cards} (2) + сравнительная таблица → {eco-map} (7) → {price-models} + таблица → {rescue-cards} (4) → {staff-panel} → {steps} (3) → FAQ (10). Сравнение построено как React Native vs нативная разработка (главная альтернатива по выдаче), не vs CMS. Риск каннибализации: лендинг таргетирует запросы, пересекающиеся со страницами услуг /uslugi/mobilnaya-razrabotka/, /uslugi/krossplatformennaya-razrabotka/, /uslugi/ios-razrabotka/, /uslugi/android-razrabotka/ — но разводятся интентом: страницы услуг продают УСЛУГУ («сколько стоит мобильная разработка», «нативное приложение»), лендинг продаёт ЭКСПЕРТИЗУ В СТЕКЕ («разработка на React Native») и распределяет посетителя по этим услугам ссылками из матрицы. Каждая строка матрицы ведёт на свою /uslugi/, поэтому лендинг работает как хаб, а не конкурент. Ссылки на ios-razrabotka/android-razrabotka даны только в FAQ про выбор стека (как «когда НЕ RN»), чтобы не перетягивать RN-трафик.
