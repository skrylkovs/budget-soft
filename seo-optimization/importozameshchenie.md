- **Страница:** `importozameshchenie/index.html` (https://www.budget-soft.ru/importozameshchenie/)
- **Дата внедрения:** 20.07.2026 (сбор данных — 19.07.2026)
- **Артефакты цикла:**
  ядро [seo-core/importozameshchenie.md](../seo-core/importozameshchenie.md) →
  контент [seo-pages/importozameshchenie.md](../seo-pages/importozameshchenie.md) →
  этот отчёт.
- **Тип страницы:** depth-1 (страница верхнего уровня), **не** `uslugi/<slug>/`.
  Собирается списком `pages_depth1` в `main()` [_generate_pages.py](../_generate_pages.py).
- **URL не менялся.** В `urls.json` обновлены только `title` и `lastmod`.

### Третья итерация: блок «Почему мы» карточками с иконками

Замечание: блок не соответствовал страницам услуг. Там «Почему мы» — это **вводная
полоса плюс отдельные карточки-преимущества**, каждая со своим `###`-заголовком и
иконкой из `images/why/`; у нас был один список из пяти пунктов.

Переделано: `{section: Почему мы}` + вводный заголовок с лидом, затем четыре карточки
(«Права остаются у вас», «Прогресс виден», «Мы разработчики, а не перепродавцы
лицензий», «Гарантия после запуска») парами 2×2 через `SERVICE_BAND_ROWS`, иконки — через
новую запись `SERVICE_BAND_ICONS_BY_SLUG["importozameshchenie"]`. Кейс SAP → 1С:ERP
перенесён во вводный абзац. «Что может пойти не так» осталась полноширинной полосой:
пять пунктов с длинными пояснениями в половину ширины не читаются.

Замеры: 4 карточки с иконками, пути `../images/why/…` корректны для depth-1, файлы на
месте, битых картинок 0; перекос новых пар 27 px и 38 px (норма <80).
**Паритет по картинкам достигнут: 23 против 23 у CRM-страницы.**

**Грабли на будущее:** Chrome кэширует `page.css`, и правка стилей может «не примениться»
при проверке — замер показывал прежние 0 px, хотя правило лежало в файле. Проверять стили
только с отключённым кэшем (CDP `Network.setCacheDisabled`).

## Результаты верификации

Программная проверка сгенерированного HTML — все пункты пройдены:

- Title / Description / Keywords на месте; canonical `https://www.budget-soft.ru/importozameshchenie/`; `noindex` нет.
- Ровно **один** H1, и он по теме страницы.
- 4 ценовые карточки; таблица `compare--service` с `compare__head` и 14 `data-label`; 3 spec-карточки; бенто-ряд получил `--first`/`--last`.
- Все `.section-lead` — по одному на `.section-head` и ≤ 250 символов.
- JSON-LD парсится как JSON; типы: Organization, WebSite, BreadcrumbList, WebPage, Service (с AggregateOffer `lowPrice: 90000`), FAQPage.
- **FAQ 1:1:** 10 видимых вопросов = 10 в FAQPage, расхождений 0, аккордеон с общим `name`.
- Браузер: HTTP 200, JS-ошибок нет, на ширине 320 px горизонтального скролла нет (`scrollWidth = clientWidth = 305`).

**Дисциплина sitemap (правило CLAUDE.md):** в `sitemap.xml` изменилась ровно одна дата —
`/importozameshchenie/` 2026-07-19 → 2026-07-20. Соседние страницы (проверено на
`/garantii/`) сохранили прежний `lastmod`. Остальные строки diff — три новых URL,
добавленных параллельной работой над другими услугами.

## Сильнейшая находка цикла — отдельная страница

**«Внесение ПО в реестр отечественного ПО (Минцифры)»** — суммарный спрос по вариациям
(383 + 296 + 246 + 240 + 139 по данным Подбора запросов) **кратно превышает** спрос
самой темы импортозамещения (95). Аудитория другая — вендор, который заводит свой
продукт в реестр, а не заказчик, уходящий с зарубежного ПО. Это кандидат на отдельную
страницу услуги, зафиксирован в ядре.

## Изменение контекста проекта

Зафиксировано впервые в этом цикле: **www-зеркало теперь подтверждено в Вебмастере**
(`https://www.budget-soft.ru`, 57 страниц добавлено, 9 в поиске). Прежний блокер из
