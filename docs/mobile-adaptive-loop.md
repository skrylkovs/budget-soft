## 5. Критерии приёмки

Каждый — бинарный: **PASS / FAIL**. Severity: **blocker** ломает пользование, **major** портит вид или доступность, **minor** косметика.

### C1. Нет горизонтального переполнения — blocker

Ни один элемент не выходит за правую границу вьюпорта.

⚠️ **`body { overflow-x: hidden }` в [styles.css:75](../styles.css#L75) маскирует переполнение** — проверка через `documentElement.scrollWidth` даст ложный PASS. Проверять поэлементно:

```js
// PASS ⟺ вернул пустой массив
Array.from(document.querySelectorAll('body *')).filter(el => {
  const r = el.getBoundingClientRect();
  if (r.width === 0 || r.height === 0) return false;
  const cs = getComputedStyle(el);
  if (cs.visibility === 'hidden' || cs.display === 'none') return false;
  if (cs.position === 'fixed') return false;              // фиксированные — отдельно, C9
  // элемент внутри легитимного горизонтального скроллера — не дефект
  let p = el.parentElement;
  while (p && p !== document.body) {
    const pcs = getComputedStyle(p);
    if (pcs.overflowX === 'auto' || pcs.overflowX === 'scroll') return false;
    p = p.parentElement;
  }
  return r.right > window.innerWidth + 1 || r.left < -1;
}).map(el => ({
  sel: el.tagName.toLowerCase() + (el.className ? '.' + String(el.className).trim().split(/\s+/).join('.') : ''),
  right: Math.round(el.getBoundingClientRect().right),
  vw: window.innerWidth
}));
```

Дополнительно: свайп-жест вправо на 320px не открывает пустую зону (визуальная проверка скриншотом).

### C2. Тап-таргеты — major

На вьюпортах < 768px каждый интерактивный элемент (`a`, `button`, `input`, `select`, `textarea`, `[role=button]`, `[onclick]`) имеет фактическую кликабельную область **≥ 44 × 44 CSS px** (Apple HIG / WCAG 2.5.5).

Допустимые исключения — фиксируются списком, не «по ощущению»:
- инлайн-ссылки внутри абзаца прозы (`p a`, `li a` в текстовых блоках) — минимум **24 × 24** (WCAG 2.5.8 AA);
- ссылки в подвале при вертикальной раскладке — минимум 44px по высоте, ширина по контенту.

Расстояние между центрами соседних тап-таргетов — **≥ 44px**; между краями — **≥ 8px**.

```js
Array.from(document.querySelectorAll('a,button,input,select,textarea,[role=button],[onclick]'))
  .map(el => ({ el, r: el.getBoundingClientRect() }))
  .filter(({el, r}) => r.width > 0 && r.height > 0 && !el.closest('p, li'))
  .filter(({r}) => r.width < 44 || r.height < 44)
  .map(({el, r}) => ({ sel: el.tagName + '.' + el.className, w: Math.round(r.width), h: Math.round(r.height) }));
```

### C3. Типографика — major

На вьюпортах < 768px:
- основной текст (`p`, `li`, ячейки таблиц) — `font-size` **≥ 16px**;
- вторичный текст (подписи, сноски, бейджи) — **≥ 13px**, и таких элементов не больше 10% текстовых узлов;
- `line-height` основного текста — **≥ 1.4**;
- длина строки — **≤ 75 символов** (при 16px это ≈ 600px ширины текстового блока);
- поля ввода (`input`, `textarea`, `select`) — `font-size` **≥ 16px**, иначе iOS Safari зумит страницу при фокусе (**blocker**, а не major).

### C4. Текст не обрезан и не наезжает — blocker

Для каждого блока с текстом: `el.scrollWidth <= el.clientWidth + 1`, **кроме** элементов с явным `text-overflow: ellipsis`, `-webkit-line-clamp` или родителем-скроллером.

Заголовки (`h1`–`h3`) на 320px не разрываются посреди слова и не вылезают за контейнер: включён `overflow-wrap: anywhere` либо `hyphens: auto` для длинных технических слов («импортозамещение», «микросервисная»).

Ни одна пара соседних элементов не перекрывается: пересечение bounding-box у элементов без `position: absolute/fixed` — FAIL.

### C5. Медиа — major

- Каждый `img` / `video` / `svg` / `canvas`: `max-width: 100%`, `height: auto` (или заданное соотношение) — не выходит за контейнер.
- У `img` заданы `width`/`height` или `aspect-ratio` — нет сдвига layout при загрузке (**CLS ≤ 0.1**).
- `hero-background.webm` (2.2 МБ) **не загружается** на вьюпортах < 900px — только `hero-poster.jpg`. Проверка: в `performance.getEntriesByType('resource')` нет записи `.webm`.
- Растровые картинки на 390×844 @ DPR 3 не выглядят мыльными: натуральная ширина ≥ ширины отрисовки × 2.

### C6. Таблицы — blocker

Таблица `compare--service` присутствует на **60 страницах услуг** и в разметке не имеет обёртки со скроллом.

Требование на вьюпортах < 640px — одно из двух, единообразно по всему сайту:
- **вариант A**: обёртка `overflow-x: auto` + видимый индикатор прокрутки (тень у края или явная подпись), при этом сама обёртка не создаёт переполнения страницы;
- **вариант B**: перестроение в карточки — каждая строка становится блоком «label → значение».

**Вариант B предпочтителен** для `compare--service`: у неё всего 3 колонки (label / legacy / ai), карточки читаются лучше горизонтального скролла.

Правка структуры — в `_generate_pages.py` (рендер таблицы, ~строка 9167 и рядом), затем регенерация всех затронутых страниц.

### C7. Навигация и меню — blocker

На вьюпортах < 900px:
- бургер видим, тап-таргет ≥ 44×44, открывает меню;
- при открытом меню фон **не скроллится** (lock на `body`), а само меню скроллится, если пунктов больше, чем помещается;
- меню закрывается: тапом по крестику, тапом по фону, клавишей Esc;
- меню не перекрыто ничем по `z-index` и полностью помещается в вьюпорт при 320×568 **и** при 844×390 (landscape);
- подменю «Услуги» (16 категорий) на мобильном разворачивается аккордеоном, а не выпадает по hover — hover-only навигация на тач-устройствах недопустима;
- активный пункт подсвечен.

### C8. Формы и модалка — blocker

Модалка «Обсудить проект» на 320×568 и на 844×390:
- помещается целиком либо скроллится внутри себя, не обрезая кнопку отправки;
- крестик закрытия в зоне большого пальца, ≥ 44×44;
- поля ≥ 16px (см. C3);
- у полей корректные `type` / `inputmode` / `autocomplete` (`tel`, `email`) — вызывается нужная клавиатура;
- при фокусе в поле и поднятой клавиатуре (эмуляция: высота вьюпорта 844 → 400) кнопка отправки достижима;
- фокус залочен внутри модалки, при закрытии возвращается на кнопку-триггер.

### C9. Фиксированные элементы и safe-area — major

- Липкая шапка занимает **≤ 15% высоты вьюпорта** в мобильном landscape (844×390 → ≤ 58px).
- Все `position: fixed` элементы учитывают вырез и домашний индикатор: `padding` через `env(safe-area-inset-top/bottom/left/right)`.
- Плавающие кнопки не перекрывают контент и не залезают на подвал.
- Сумма фиксированных элементов не съедает > 25% высоты вьюпорта.

### C10. Отступы и сетка — minor

- Горизонтальный padding контейнера: **≥ 16px** при < 480px, **≥ 24px** при 480–768px.
- Ни один контентный блок не прилипает к краю экрана вплотную (кроме намеренно полноширинных — hero, полосы клиентов).
- Сетки не оставляют «сирот»: последняя карточка в ряду не растягивается на всю ширину, если остальные — половинные (либо это осознанное правило).
- Перекос бенто-пар < 80px (существующая норма проекта).

### C11. Анимации и производительность — major

- На вьюпортах < 768px canvas-фоны (`pixel-blast.js`, `radar.js`, `soft-aurora.js`, `logo-loop.js`) отключены либо деградированы — они не должны стартовать RAF-цикл. Проверка: FPS при скролле ≥ 50.
- `@media (prefers-reduced-motion: reduce)` уважается всеми анимациями, включая canvas и `case3d.js`.
- `case3d.js` (3D-наклон по mousemove) на тач-устройствах отключён.
- Мобильный Lighthouse Performance для `/` и одной страницы услуги — **≥ 70**, CLS ≤ 0.1, LCP ≤ 4с (throttling: Slow 4G).

### C12. Отсутствие десктопной регрессии — blocker

После каждой итерации скриншоты 1440×900 и 1920×1080 для всех страниц-эталонов сравниваются с состоянием до итерации: **различия только в намеренно изменённых зонах**. Любое неожиданное расхождение > 2% пикселей блока — FAIL, правку откатить и переделать более узко.

### C13. Консоль чиста — major

На каждом вьюпорте: ноль ошибок в консоли, ноль 404 в сетевых запросах.

## 6. Порядок итерации

1. Поднять `python -m http.server 8000` (если не поднят).
2. **Аудит**: прогнать матрицу §2 по эталонам §4, собрать дефекты в JSON `{page, viewport, criterion, severity, selector, detail}`.
3. Сохранить сырой отчёт в `docs/mobile-audit/<итерация>.json`.
4. **Приоритизация**: blocker → major → minor; внутри — по числу затронутых страниц (дефект в общем блоке шапки/подвала бьёт по 136 страницам, он важнее уникального).
5. **Починка**: брать 3–7 дефектов за итерацию, не больше. Правки — по I2/I3: CSS в `styles.css`/`page.css`, структура в `_generate_pages.py` + регенерация.
6. **Переверификация**: перепрогнать те же страницы/вьюпорты. Дефект исчез И новых не появилось И C12 чист — иначе откат.
7. Обновить `docs/mobile-audit/progress.md`: что починено, чем проверено, что осталось.
8. Когда тир A чист по эталонам — подключить тир B, затем сквозной тир C по всем 136 страницам.

## 7. Стоп-условие

Цикл останавливается, когда **одновременно**:

- [ ] 0 дефектов severity **blocker** и **major** на тирах A и B по всем страницам-эталонам;
- [ ] 0 дефектов C1/C2/C4/C5 на тире C (все 136 страниц × 320/390/768);
- [ ] C12 чист — десктоп 1440/1920 без непреднамеренных расхождений;
- [ ] Lighthouse mobile ≥ 70 на `/` и на странице услуги;
- [ ] **две итерации подряд** не нашли ни одного нового дефекта blocker/major;
- [ ] `progress.md` содержит финальную сводку: сколько дефектов найдено/починено, какие minor осознанно оставлены и почему.

Если дефект не чинится за 3 итерации — записать в `progress.md` как «заблокировано», указать причину, идти дальше. Не крутиться на нём бесконечно.

## 8. Известные грабли проекта

- Синтетический `hover` в headless флакует (дрейф координат, canvas на главной) — проверять hover-инварианты через CSS-состояние, а не координатный наведённый курсор; делать ретраи.
- Параллельные сессии ломают генератор посреди правок — перед регенерацией убедиться, что никто больше не пишет в `_generate_pages.py`.
- Общий профиль Playwright между сессиями — см. §1.
- `lowPrice` захардкожен, адрес офиса — константа (Пресненская наб., 12). Контент не трогать: задача чисто про адаптивность.
