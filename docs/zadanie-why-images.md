## Общие требования к генерации

Базовый шаблон промпта — проверенный (им сгенерирован текущий комплект):

```
нарисуй 4 картинки размером 100×100 с прозрачным фоном
цветовая гамма - темная
стиль - минималистичный, cyberpunk
```

Дополнения к шаблону (важны для серии):

- в каждый промпт добавлена строка «тематика страницы …» — чтобы картинки
  разных страниц отличались, ведь 2 из 4 карточек («60+ проектов за 17 лет»,
  «До 4× дешевле») повторяются на всех страницах;
- надписи внутри картинок — только крупные и короткие («60+», «4×», «MVP»);
  кириллицу генераторы часто ломают — после генерации проверить каждую букву,
  при артефактах перегенерировать или просить вариант без текста;
- фон строго прозрачный (картинка лежит на белой карточке);
- один доминирующий объект на картинку — при 115px мелкие детали пропадают.

# Часть 3. Иллюстрации блока «Типичные сигналы» (секция «О решении»)

Дата: 12.07.2026. Статус: ✅ ГОТОВО — все 91 страница закрыта (0 на старой `erp-signals.webp`).

## Что это за картинка

Крупная иллюстрация справа от чек-листа «Типичные сигналы, что пора …» в
секции «О решении». Выводится рендерером `make_about_body_renderer`
(`_generate_pages.py`), файл задаётся параметром `img` при регистрации
в `SECTION_BODY_RENDERERS[(slug, "О решении")]`; по умолчанию —
`erp-signals.webp`. Слот в CSS: `.erp-map__signals-visual`,
`aspect-ratio: 380/330` + `object-fit: cover`.

**Важно про размер.** Картинки генерируются 400×300, но слот имеет пропорцию
380/330 — при `cover` у 400×300 срезаются левый и правый края (~27px с каждой
стороны). Поэтому перед установкой картинки **дополняются прозрачными полями
сверху/снизу до 400×348** (=380/330) — тогда иллюстрация видна целиком и
масштаб одинаков на всех страницах. Без этого у «широких» композиций
(SCM, WMS, 1С, ИИ, спасение проектов) обрезался край.

```python
# дополнение полями перед копированием в images/
canvas = Image.new('RGBA', (400, 348), (0, 0, 0, 0))
canvas.paste(im, ((400 - im.width) // 2, (348 - im.height) // 2), im)
```
