# План MVP: AI-калькулятор стоимости разработки ПО

> Базовое ТЗ: [plan-ai-calkulyator-stoimosti-razrabotki.md](plan-ai-calkulyator-stoimosti-razrabotki.md).
> Этот документ — сжатый план **первой версии (MVP)**: что делаем, чем, как выглядит и в каком порядке.

---

## 0. Цель MVP

Запустить рабочий калькулятор, который:

1. Проводит клиента по короткой анкете (11 шагов).
2. Считает диапазон часов/срока/стоимости **по правилам** (rule-based).
3. Делит результат на **MVP / Full version**.
4. Генерирует AI-summary и список рисков (Claude API).
5. Собирает лид и отдаёт PDF.
6. Показывает менеджеру заявки в админке.

Не цель MVP: динамический конструктор вопросов, личный кабинет клиента, оплата, CRM-интеграции, мультиязычность, lead scoring, аналитика (см. [§33 ТЗ](plan-ai-calkulyator-stoimosti-razrabotki.md)).

---

## 1. Scope MVP (in / out)

**В MVP:**
- Лендинг калькулятора + SEO-блок (H1/H2/FAQ).
- Анкета: 9–11 шагов, прогресс-бар, режим «вставить описание проекта».
- Rule-based движок оценки (типы проектов + модули + интеграции + множители).
- AI-слой: классификация проекта, summary, риски, MVP-рекомендация (Claude API, structured JSON).
- Экран результата: summary, breakdown по ролям, MVP vs Full, cost drivers, риски, дисклеймер.
- Lead capture (после 60–80% анкеты).
- PDF-отчёт.
- Email клиенту + email менеджеру.
- Админка: список лидов, карточка лида, настройка ставок и коэффициентов.

**Не в MVP:** drag-and-drop редактор вопросов, lead scoring, аналитика конверсий, мультиязычность, оплата, CRM/ERP-интеграции, личный кабинет.

## 2. Архитектура и стек

Соответствует §29 ТЗ, минимальный набор для MVP:

| Слой | Технология |
| --- | --- |
| Backend | Laravel 12, PHP 8.3 |
| БД | PostgreSQL (или MySQL) |
| Очереди/кеш | Redis (AI-вызовы и email — в очередь) |
| Frontend (калькулятор) | React + Inertia (в составе Laravel-приложения) |
| Стили | Tailwind CSS + дизайн-токены «Revolut-style» (см. §3) |
| Админка | Filament (тема перекрашена под §3) |
| AI | Claude API (`claude-opus-4-8` для анализа ТЗ, `claude-haiku-4-5` для дешёвых классификаций) |
| PDF | Browsershot (Puppeteer) — HTML-шаблон → PDF |
| Email | SMTP / Mailgun, отправка через queue |

> Стек самодостаточен (Laravel + Inertia + Filament) — не нужен отдельный Next.js, что сокращает MVP.

## 3. Дизайн-система «в стиле Revolut Business»

> ⚠️ Токены смоделированы по фирменному стилю revolut.com/business (точные hex/шрифты откалибровать по живому сайту, когда будет доступ). Шрифт Revolut «Aeonik» — коммерческий; в MVP берём близкий бесплатный гротеск.

**Характер стиля Revolut Business:**
- Чисто, дорого, много воздуха; крупная жирная типографика.
- Контраст «глубокий тёмный + белый», акцент — электрик-синий.
- Секции с мягкими пастельными фонами (мята, лаванда, персик) чередуются с тёмными.
- Скруглённые карточки и «пилюли»-кнопки, мягкие тени, тонкие бордеры.

**Токены (CSS custom properties):**

```css
:root {
  /* Цвета */
  --rv-black:      #0A0B0D;   /* фон тёмных секций, основной текст */
  --rv-ink:        #191C1F;   /* заголовки */
  --rv-white:      #FFFFFF;
  --rv-blue:       #1B1BFF;   /* primary accent (электрик-синий) */
  --rv-blue-600:   #0C53FF;   /* hover */
  --rv-gray-500:   #6B7177;   /* вторичный текст */
  --rv-gray-100:   #F4F5F6;   /* фон светлых карточек */
  --rv-line:       #E6E8EA;   /* бордеры */
  /* Пастельные фоны секций */
  --rv-mint:       #DDF3E4;
  --rv-lavender:   #E7E3FB;
  --rv-peach:      #FCE8DD;

  /* Типографика — Aeonik → запасной бесплатный гротеск */
  --rv-font: "Aeonik Pro", "Hanken Grotesk", "Manrope", "Inter", system-ui, sans-serif;

  /* Радиусы / тени / шаги сетки */
  --rv-radius:     20px;   /* карточки */
  --rv-radius-pill: 999px; /* кнопки */
  --rv-shadow:     0 8px 30px rgba(10,11,13,.08);
  --rv-gap:        8px;    /* база spacing-сетки (8/16/24/40/64) */
}
```

**Компоненты:**
- **Кнопка primary:** `--rv-black` фон, белый текст, `border-radius: var(--rv-radius-pill)`, padding `16px 28px`, без тени; hover → `--rv-blue`.
- **Кнопка secondary:** прозрачная, бордер `1px var(--rv-ink)`, чёрный текст.
- **Карточка вопроса/опции:** белый фон, `--rv-radius`, бордер `1px --rv-line`, при выборе → бордер `2px --rv-blue` + лёгкий тинт.
- **Заголовки:** жирные (600–700), крупный hero `clamp(40px, 6vw, 80px)`, плотный letter-spacing `-0.02em`.
- **Прогресс-бар:** тонкая линия `--rv-blue` поверх `--rv-gray-100`.
- **Секции лендинга:** чередование белый ↔ пастель ↔ тёмный.

**Применяется к двум поверхностям:**
1. **Калькулятор (клиент)** — лендинг + анкета + результат.
2. **Админка (Filament)** — перекрашенная тема: те же шрифт, акцент-синий, скругления, «пилюли»-кнопки.

## 4. Flow калькулятора (экраны)

11 шагов (§34 ТЗ). Один вопрос — один экран, прогресс-бар сверху.

```
Step 0  Лендинг (hero + «Оценить за 3 минуты» + выбор режима: анкета / вставить ТЗ)
Step 1  Тип проекта
Step 2  Стадия проекта
Step 3  Платформы
Step 4  Роли пользователей
Step 5  Ключевые функции (модули)
Step 6  Интеграции
Step 7  Дизайн-статус
Step 8  AI-функции
Step 9  Сроки и бюджет
Step 10 Контакты (lead capture)  ← после ~70% анкеты
Step 11 Результат (summary + breakdown + MVP/Full + риски + PDF + CTA)
```

Альтернативный вход (режим «у меня есть описание»): textarea → AI разбирает → префилл шагов 1–8 → клиент подтверждает/правит → результат.

Состав вопросов и варианты ответов — §35 ТЗ.

## 5. Движок расчёта (rule-based)

Конфиг хранится в БД/JSON (seed), редактируется админом. Формула — §18–19 ТЗ:

```
total = base(project_type)
      + Σ module_hours
      + Σ integration_hours
      + design + qa + devops + pm
total_adjusted = total × complexity × risk
final = total_adjusted + buffer(risk)
```

Выход разбивается по ролям (Backend/Frontend/QA/DevOps/PM/Design/BA) и на MVP vs Full.
Стоимость = часы × ставка из rate card (режимы Economy/Standard/Premium).
Показываем **диапазон**, не точку, + уровень уверенности (±20/35/50%).

Сидируем для MVP: 8 типов проектов, 20–30 модулей, базовые интеграции, множители сложности/риска.

## 6. AI-слой (Claude API)

AI **не считает цену** — только классифицирует и объясняет (§26–28 ТЗ).

- **Вход:** ответы анкеты + (опц.) текстовое описание.
- **Промпт:** §27 ТЗ, structured JSON output.
- **Выход (JSON, §28):** `project_type`, `modules`, `integrations`, `ai_features`, `complexity_level`, `risk_level`, `missing_information`, `mvp_recommendation`, `estimation_notes`.
- Этот JSON → кормит rule-based движок (модули/множители) и текстовые блоки результата (summary, риски, рекомендация).
- Вызовы — в очередь (Redis), с фолбэком: если AI недоступен, показываем чистый rule-based результат без summary.

## 7. База данных (MVP-минимум)

Из §30 ТЗ берём 6 таблиц:

- `estimation_sessions` — сессия расчёта (тип, сложность, риск, диапазоны часов/цены).
- `estimation_answers` — ответы анкеты.
- `estimation_modules` — выбранные модули с часами по ролям.
- `estimation_leads` — контакты + статус (New/Contacted/Qualified/Lost).
- `estimation_settings` — коэффициенты/множители.
- `estimation_rate_cards` — ставки по ролям.

## 8. API (MVP)

Public (§31 ТЗ):
```
POST /api/estimation/session/start
POST /api/estimation/session/{uuid}/answer
POST /api/estimation/session/{uuid}/analyze     ← AI
GET  /api/estimation/session/{uuid}/result
POST /api/estimation/session/{uuid}/lead
GET  /api/estimation/session/{uuid}/pdf
```
Admin — через Filament (resources), отдельный REST не обязателен в MVP.

## 9. PDF и Email

- **PDF** (§23 ТЗ): логотип, тип проекта, оценка часов/стоимости, разбивка по ролям, MVP/Full, модули, риски, рекомендация, CTA «бесплатная консультация». HTML-шаблон в стиле §3 → Browsershot.
- **Email клиенту** (§41): краткая оценка + ссылка на PDF.
- **Email менеджеру** (§42): данные лида + AI-summary + риски + ссылка в админку.

## 11. Метрики, риски, дисклеймер

- **Метрики MVP (минимум):** старт анкеты, завершение, оставлен email, скачан PDF.
- **Дисклеймер** (§37) — обязателен на экране результата и в PDF.
- **Риски продукта** (§46): заниженная оценка → диапазоны + буфер; высокая цена пугает → MVP отдельно; AI врёт → расчёт rule-based, AI только текст; длинная анкета → 9–11 шагов + прогресс.

## 13. Definition of Done (MVP)

- [ ] Клиент проходит анкету и видит диапазон часов/срока/стоимости с разбивкой MVP/Full.
- [ ] AI-summary и риски генерируются (с фолбэком при сбое AI).
- [ ] Лид сохраняется, менеджеру уходит письмо, клиенту — PDF.
- [ ] Менеджер видит лид в админке, меняет статус, видит ответы и расчёт.
- [ ] Админ меняет ставки и коэффициенты без кода.
- [ ] Калькулятор и админка выдержаны в Revolut-style дизайн-системе.
